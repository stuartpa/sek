Spec Explorer 2010 Samples

For Version 3.5.3146.0

2013-6-27

This is a collection of samples for SE 2010. Every sample has a README available as a solution item which includes for running the sample.

Click on the sample header to navigate to the samples directory, and open the solution file to run the sample.

Account

This sample demonstrates how to use reference types as parameters of static actions. The TypeBinding attribute is used to establish the mapping between a modeling type and an implementation type. Spec Explorer handles the mapping between model instances and implementation instances automatically at test runtime.

For an example of how to declare instance-based method as actions, select the corresponding option in the Spec Explorer model project wizard (visual Studio File menu -> New -> Project -> Visual C# -> Test -> Spec Explorer Model).

ATSVC

This sample demonstrates:

-          Modeling of a simple system service

Note: you must run VS with administrator privileges to successfully execute the test suites in this sample.

CHAT

This sample represents a "fake" protocol, called MS-CHAT. It demonstrates some modeling patterns and the usage of BinaryFormatter to automatically marshal and unmarshal bits to the wire. The protocol is a simple chat server which allows user to logon, logoff, gets a list of logged-on users, and finally to publish a message which will be broadcasted to all logged-on users. The protocol is block-oriented and runs over TCP. A fake server is provided as well.

This sample also shows how to use Requirement Coverage Machine with combination of switches, RequirementsToCover(set of requirements) and MinimumRequirementsCount(minimun requirements covered in the generated graph)

Operators

This is a simple sample to show the usages of all operators in Cord.

Parameter Generation

This solution contains some samples how to use new parameter combination features. The samples match the ones used in the Parameter Generation Guide.

Note that the samples are not representing any real system or protocol, and no implementation or test suite is generated. The samples just serve for demonstrating features of parameter generation.

PubSub

This is an SE sample which demonstrates instance based modeling using the simple pattern of a publish-subscriber managed API.

Requirement Report

This is a simple sample to show the usages of customized post-processing.

Users can copy RequirementReport.dll into Extensions folder under Spec Explorer installation path, and launch this processing from Exploration Manager to generate requirement report for any machine. The default output folder is the model project directory

Sailboat

This is a sample for the Point & Shoot exploration strategy.

See the Point & Shoot user guide in help file for explanations.

SMB2

This is a SIMPLIFIED version of model and adapter for SMB2 which captures some aspects of the protocol. This model is not intended to solve the SMB2 test suite problem, but only for demonstrating certain concepts. The sample is of larger complexity and shows various modeling patterns and adapter implementation patterns.
